redux和vuex和pinia ​

Redux： Redux 是一个独立于框架的状态管理库，最初是为 React 设计的，但可以在其他框架中使用。它的核心概念包括单一的全局状态树、纯函数来处理状态变化以及通过 action 来触发状态的变化。

  1. action：一个js对象，包含两个属性：
    1. type： 标识属性，值是字符串。多个type用action分开
    2. payload：数据属性，可选。表示本次动作携带的数据
  2. reducer
  3. store:仓库，Redux 的核心，整合 action 和 reducer

vuex的五大部分 ​

  1. State（状态）：State 是应用程序的中央数据存储库，代表了应用程序的整体状态。
  2. Mutation（变更）：Mutation 是修改 State 的唯一途径。
  3. Action（动作）：Action 类似于 Mutation，不同之处在于 Action 是用于执行异步操作或批量的 Mutation。
  4. Getter（获取器）：Getter 用于从 State 中派生出一些衍生状态，类似于组件中的计算属性。
  5. Module（模块）：Module 可以将 Store 分割成多个小模块，每个模块都有自己的 State、Mutation、Action、Getter，并且可以嵌套子模块。

pinia ​

  1. store
  2. State（状态）
  3. Actions（动作）
  4. Getters（获取器） Mutations没有了，用this获取数据

Redux ​

Redux是一个可预测的状态容器

  1. 状态（State）
  2. 容器（Container）
  3. 可预测（Predictable）

Redux是一个状态容器，所以使用Redux必须先创建容器对象，它的所有操作都是通过容器对象来进行的，createStore用来创建一个Redux中的容器对象，它需要三个参数：reducer、preloadedState、enhancer。

js
const store = Redux.createStore(reducer, [preloadedState], [enhancer])
  • reducer是一个函数，是state操作的整合函数，每次修改state时都会触发该函数，它的返回值会成为新的state。
  • preloadedState就是state的初始值，可以在这里指定也可以在reducer中指定。
  • enhancer增强函数用来对state的功能进行扩展

store常用的方法 ​

  • createStore可以帮助创建 store
  • store.dispatch 帮助派发 action , action 会传递给 store
  • store.getState 这个方法可以帮助获取 store 里边所有的数据内容
  • store.subscrible 方法订阅 store 的改变，只要 store 发生改变， store.subscrible 这个函数接收的这个回调函数就会被执行
js
store.getState()  //获取当前应用的状态（state）

store.dispatch({type:'ADD'})  //例如派发 action 来修改状态

store.subscribe(()=>{
    // 方法订阅 store 的改变,你可以在 Redux 中监听状态的变化，并在状态发生变化时执行相应的操作
});

多个reducer的Redux用法 ​

  1. 一个复杂的reducer分成几个reducer，最后用combineReducers合并
  2. 创建reducer的参数state可以写默认值
  3. 创建store后，需要引入react-redux中提供的Provider组件，将其设置到所有组件的最外层，并且将刚刚创建的store设置为组件的store属性，只有这样才能使得Redux中的数据能被所有的组件访问到。
  4. react-redux还为我们提供一个钩子函数useSelector，用于获取Redux中存储的数据，它需要一个回调函数作为参数，回调函数的第一个参数就是当前的state，回调函数的返回值，会作为useSelector的返回值返回
js
import ReactDOM from 'react-dom/client';
import {Provider, useDispatch, useSelector} from "react-redux";
import {combineReducers, createStore} from "redux";

const stuReducer = (state = {
    name: '孙悟空',
    age: 18,
    gender: '男',
    address: '花果山'
}, action) => {
    switch (action.type) {
        case 'SET_NAME':
            return {
                ...state,
                name: action.payload
            };
        case 'SET_AGE':
            return {
                ...state,
                age: action.payload
            };
        case 'SET_ADDRESS':
            return {
                ...state,
                address: action.payload
            };
        case 'SET_GENDER':
            return {
                ...state,
                gender: action.payload
            };
        default :
            return state;
    }

};

const schoolReducer = (state = {

    name: '花果山一小',
    address: '花果山大街1号'

}, action) => {
    switch (action.type) {
        case 'SET_SCHOOL_NAME':
            return {
                ...state,
                name: action.payload
            };
        case 'SET_SCHOOL_ADDRESS':
            return {
                ...state,
                address: action.payload
            };
        default :
            return state;
    }

};

const reducer = combineReducers({
    stu:stuReducer,
    school:schoolReducer
});

const store = createStore(reducer);  //创建redux

const App = () => {
    const stu = useSelector(state => state.stu);
    const school = useSelector(state => state.school);
    const dispatch = useDispatch();
    return <div>
        <p>
            {stu.name} -- {stu.age} -- {stu.gender} -- {stu.address}
        </p>
        <div>
            <button onClick={() => {
                dispatch({type: 'SET_NAME', payload: '猪八戒'});
            }}>改name
            </button>
            <button onClick={() => {
                dispatch({type: 'SET_AGE', payload: 28});
            }}>改age
            </button>
            <button onClick={() => {
                dispatch({type: 'SET_GENDER', payload: '女'});
            }}>改gender
            </button>
            <button onClick={() => {
                dispatch({type: 'SET_ADDRESS', payload: '高老庄'});
            }}>改address
            </button>
        </div>

        <hr/>

        <p>
            {school.name} -- {school.address}
        </p>
        <div>
            <button onClick={() => {
                dispatch({type: 'SET_SCHOOL_NAME', payload: '高老庄小学'});
            }}>改学校name
            </button>
            <button onClick={() => {
                dispatch({type: 'SET_SCHOOL_ADDRESS', payload: '高老庄中心大街15号'});
            }}>改学校address
            </button>
        </div>
    </div>;
};


const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
    <div>
        <Provider store={store}>
            <App/>
        </Provider>
    </div>
);

redux的中间件 ​

中间件的作用： 就是在 源数据 到 目标数据 中间做各种处理，有利于程序的可拓展性，通常情况下，一个中间件就是一个函数，且一个中间件最好只做一件事情。 调用 dispatch 触发 reducer之前做一些其他操作，也就是说，它改变的是执行dispatch到 触发 reducer的流程。

Redux 提供了一个 applyMiddleware 函数，可以用于使用中间件。

  1. applyMiddleware的内部实现：总体来说就是接收外部传入的createStore、reducer、preloadedState参数，用createStore生成一个新的store对象，对新store对象中的dispatch方法用中间件增强，返回该store对象。
  2. 复合函数，也就是洋葱模型，compose(A, B, C)的返回值是：(arg) => A(B(C(arg)))，内函数的输出就是外函数的输入，即 action 会先经过第一个中间件，然后依次执行下一个中间件，再最后到达 reducer。这样dispatch作为参数会经过所有中间件，返回增强后的dispatch

redux applyMiddleware.js ​

js
//

export default function applyMiddleware(...middlewares) {
  return (createStore) => (reducer, preloadedState, enhancer) => {
    const store = createStore(reducer, preloadedState, enhancer)
    let dispatch = store.dispatch
    let chain = []

    const middlewareAPI = {
      getState: store.getState,
      dispatch: (action) => dispatch(action)
    }
    chain = middlewares.map(middleware => middleware(middlewareAPI))
    dispatch = compose(...chain)(store.dispatch)

    return {
      ...store,
      dispatch   // 返回给全局store的是经过中间件增强的dispatch
    }
  }
}

const compose = (...funcs) => {
  return funcs.reduce((f, g) => (x) => f(g(x)));
}

第三方中间件 ​

js
import { createStore, applyMiddleware } from 'redux';
import thunkMiddleware from 'redux-thunk'; //异步处理中间件
import loggerMiddleware from 'redux-logger';//日志功能
import rootReducer from './reducers'; // 导入你的 rootReducer


const middleware = [thunkMiddleware, loggerMiddleware]; // 将需要的中间件放入数组中

const store = createStore(
  rootReducer,
  applyMiddleware(...middleware)
);

自定义中间件next(action);调用下一个中间件或 reducer ​

js
const middleware1 = store => next => action => {
  console.log('Middleware 1: Before action is dispatched');
  next(action); // 调用下一个中间件或 reducer
  console.log('Middleware 1: After action is dispatched');
};

const middleware2 = store => next => action => {
  console.log('Middleware 2: Before action is dispatched');
  next(action); // 调用下一个中间件或 reducer
  console.log('Middleware 2: After action is dispatched');
};

const middleware3 = store => next => action => {
  console.log('Middleware 3: Before action is dispatched');
  next(action); // 调用下一个中间件或 reducer
  console.log('Middleware 3: After action is dispatched');
};

// 组合中间件
const chain = [middleware1, middleware2, middleware3];

// 创建 Redux Store 时，使用 applyMiddleware 应用中间件
const store = createStore(
  rootReducer,
  applyMiddleware(...chain)
);

// 派发一个 action
store.dispatch({ type: 'SOME_ACTION' });

函数柯里化（Currying） ​

嵌套函数，相当于c(b(a()))，a的返回值传给b，b的返回值传给c

js
const fun = a => b => c => {
  console.log(`a: ${a}, b: ${b}, c: ${c}`);
}; 

//相当于c(b(a()))

const fn1 = function(a){
    console.log(a);   // Hello
    return function(b){
        console.log(a,b);  // Hello world
        return function(c){
            console.log(a,b,c);  // Hello world !
        }
    }
}

// 使用这个函数，先传递 a，再传递 b，最后传递 c
fn1('Hello')('world')('!'); // 输出：a: Hello, b: world, c: !
